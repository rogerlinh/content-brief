## 2. KORAY TOPICAL AUTHORITY FRAMEWORK

### 2.1 Core Formula

```
Topical Authority = Historical Data × Topical Coverage
```

**Key principle:** Google wants to rank the website that is **CHEAPER to crawl** and **CERTAINTY to rank**.

- Website ranking = cost of crawler budget vs. cost of not ranking
- High page rank + no traffic = NO authority
- Quality Nodes (top articles) must be placed on homepage for crawler proximity

### 2.2 SCN = Semantic Content Network

```
SCN = Topical Map + Knowledge Base
```

- **Topical Map:** Boundaries between topics based on entity proximity and attributes
- **Knowledge Base:** Facts + entity relationships (EAV triples)

### 2.3 The 6 Pillars of Koray Framework

Every content piece must satisfy all 6:

1. **Contextual Vector** — Heading flow reflects main topic progression
2. **Contextual Hierarchy** — H1 → H2 → H3 → H4 logical parent-child relationships
3. **Contextual Structure** — Main Content vs. Supplemental Content split
4. **Contextual Connection** — Internal links with justified anchor text
5. **Contextual Coverage** — Proper depth per topic based on importance
6. **Contextual Flow** — Seamless transitions, circular flow (H1 connects to ending)

### 2.4 Publishing Frequency

> **Most important element of Topical Authority.**
Without consistency, Google cannot crawl, index, and re-rank your website properly.

## 3. CONTENT BRIEF GENERATION WORKFLOW

### Step 1: Keyword Research & Source Context

For every keyword, establish:
1. **Central Entity** — the invariant topic appearing in all articles
2. **Source Context** — brand identity, USP, industry, GEO keywords, NAP (Name/Address/Phone), tone
3. **Macro Context** — main topic of the entire website
4. **Micro Context** — sub-topics, varying, often in Supplemental Content

### Step 2: Entity-Attribute-Value (EAV) Table

Build EAV triples from SERP data:

| Entity | Attribute | Value |
|--------|-----------|--------|
| Thép thanh vằn | Loại | Thanh vằn mạ kẽm |
| Thép thanh vằn | Tiêu chuẩn | JIS G3506, TCVN 1651 |
| Thép thanh vằn | Ứng dụng | Xây dựng, cơ sở hạ tầng |

**3-Pillar Attribute Filtration:**
1. **Prominence** — mandatory for entity definition
2. **Popularity** — based on search demand
3. **Relevance** — to the query's intent

### Step 3: Outline Structure (Mindshare Model)

Use the **Mindshare Model** for informational content:
```
[MAIN] Định nghĩa → [MAIN] Thuộc tính → [MAIN] Ứng dụng → [MAIN] So sánh → [SUPP] FAQ → [SUPP] Antonym Ending
```

**Main Content vs. Supplemental Content:**
- **[MAIN]** = 65-80% of content — core information answering the query
- **[SUPP]** = 20-35% — FAQ, tips, antonym endings, variations

### Step 4: SAPO (Supplementary Angle Paragraph Opening)

Each [MAIN] section opens with a **SAPO** (≥80 words):
- Direct answer first sentence (no preamble)
- Definition → Attributes → Application flow
- Brand mention naturally embedded
- Geographic + Industry context

### Step 5: Internal Linking

**Rules:**
- Internal links only deserve crawling if they have proper context
- Before linking, mention the target term 1-5 times with unique information
- Quality Nodes placed on homepage for crawler proximity
- Cluster structure: Root Article → Seeds → Quality Nodes → Content Nodes

### Step 6: Featured Snippet + PAA Mapping

- **Preceding Question** — answer the representative question first
- **Featured Snippet** — ≤60 words, direct answer format
- Map PAA questions to specific H2/H3 headings

## 4. SEMANTIC SEO WRITING RULES (12 Rules)

Apply these in every section:

### Rule 1: Proper Word Sequence
Write attribute first, not buried in clause.
- ❌ "A car, which is red and fast, sped by"
- ✅ "A fast, red car sped by"

### Rule 2: Factual Certainty
Use precise, confident language. Avoid opinion words.
- ❌ "This strategy should/will boost your traffic"
- ✅ "This strategy is designed to boost traffic by X%"

### Rule 3: Eliminate Fluff
- ❌ "It is an undeniable fact that in order to achieve success, one must work hard"
- ✅ "Hard work is essential for success"

### Rule 4: Numeric Values
Always use specific numbers.
- ❌ "Our product has many features"
- ✅ "Our product offers 7 key features"

### Rule 5: Qualify Instances
Specify severity, type, degree.
- ❌ "The disease has symptoms"
- ✅ "The disease presents 3 common symptoms: fever, cough, fatigue"

### Rule 6: Contextual Verbs
Match verbs to context:
- Health → "boosts" ("Vitamin C boosts immunity")
- Skills → "develops" ("Practice develops proficiency")
- Improvements → "enhances", "improves"

### Rule 7: Examples After Plural Nouns
- ❌ "There are various marketing tools"
- ✅ "There are various tools, such as Google Analytics, SEMrush, Ahrefs"

### Rule 8: Parallel Structure in Lists
- ❌ "To succeed: planning, to execute, reviewing"
- ✅ "To succeed: plan, execute, review"

### Rule 9: Optimize First Sentence Under Heading
First sentence must DIRECTLY answer or address the heading's topic.

### Rule 10: Answer Directness
- ❌ Start with "Trong bài viết này, chúng ta sẽ..."
- ✅ Start with the answer itself

### Rule 11: Anchor Text Context
Before internal linking, justify the link with 1-5 mentions of the term.
Never link without context.

### Rule 12: Do Not Delay Answer
Bold the answer, not the search term.

## 5. GOOGLE E-E-A-T & HELPFUL CONTENT

### E-E-A-T Requirements

| Signal | Vietnamese Steel Context |
|--------|----------------------|
| **Experience** | "Đã cung cấp 15+ năm thép xây dựng toàn quốc" |
| **Expertise** | Technical specs: JIS, ASTM, TCVN standards cited |
| **Authoritativeness** | Case studies, CO/CQ certificates, client references |
| **Trustworthiness** | Full NAP, warehouse address, tax code |

### People-First Content Checklist
- [ ] Audience finds this useful directly
- [ ] Demonstrates first-hand expertise
- [ ] Would readers trust this in a printed encyclopedia?
- [ ] Adds substantial value vs. existing top results
- [ ] Avoids "keyword stuffing" mentality
- [ ] No thin affiliate/duplicate content

### YMYL Warning
For transactional/commercial content (steel purchasing), include:
- Technical certifications
- Pricing transparency
- Safety standards compliance
- Delivery and warranty information

## 6. AI / GEO OPTIMIZATION (Google AI Overview + ChatGPT)

### How AI Extracts Content
1. AI doesn't browse — it **extracts** from structured, clear content
2. Short sentences (15-20 words) are preferred
3. LLMs struggle with long dense paragraphs

### AI Citation Checklist
- [ ] H2s and H3s clearly separate ideas
- [ ] Question-based headings for FAQ potential
- [ ] Bold key terms
- [ ] Bullets and clear formatting
- [ ] Descriptive captions for visuals
- [ ] Tables with clear headers
- [ ] Numbered lists for step-by-step processes
- [ ] "How to" / "X is Y" direct answer format

### Schema Markup to Include
- FAQPage schema for Q&A sections
- HowTo schema for procedural content
- Product schema for commercial intent
- LocalBusiness schema for GEO keywords

## 7. REVIEW WRITING STANDARDS (14 Best Practices)

When generating review comparison briefs:
1. Evaluate from user's perspective
2. Demonstrate expert knowledge
3. Show evidence: visuals, first-hand photos, test results
4. Quantitative measurements (specs comparison table)
5. What sets apart from competitors
6. Coverage for different use cases
7. Benefits AND drawbacks with research
8. Product evolution from previous models
9. Key decision-making factors with expertise evidence
10. Design choices and their user impact
11. Links to useful resources
12. Multiple vendor links for user choice
13. "Best for X" claims with first-hand evidence
14. Sufficient useful content even without affiliate pages
